# Contributing to Custom Java Collection Framework

Thank you for your interest in contributing to this project.

This repository is designed as a **learning project to understand the internal design of the Java Collection Framework**.

We welcome improvements, bug fixes, and learning enhancements.

---

## 📌 How to Contribute

1. Fork the repository

2. Clone your fork

```bash
git clone https://github.com/yourusername/custom-java-collection-framework.git
```

3. Create a new branch

```bash
git checkout -b feature/new-feature
```

4. Make your changes

5. Commit your changes

```bash
git commit -m "Added new collection feature"
```

6. Push your branch

```bash
git push origin feature/new-feature
```

7. Open a Pull Request

---

## 📂 Contribution Areas

You can contribute in the following areas:

### 1. Data Structure Implementation

Examples:

* Dynamic ArrayList implementation
* HashMap internal hashing logic
* TreeSet using Tree structure
* LinkedList node implementation

### 2. Code Improvements

* Better comments
* Clean code practices
* Performance improvements

### 3. Documentation

* Diagrams
* Architecture explanation
* Usage examples

### 4. Testing

* Unit tests
* Example programs
* Edge case testing

## 🧑‍💻 Coding Guidelines

* Use meaningful class names
* Follow Java naming conventions
* Add proper comments
* Maintain package structure
* Keep code readable

Example:

```java
public interface MyCollection extends MyIterable {

    void add(Object obj);

    void remove(Object obj);

}
```
